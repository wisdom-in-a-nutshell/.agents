# Paper Design

The live Paper MCP already provides the real tool surface. Use this skill for the extra local guidance that is easy to forget.

## Load only what you need

- Start with `references/overview.md`.
- If you are actively editing the canvas, also load `references/collaboration-workflow.md`.
- Load `references/visual-review-harness.md` when the work needs independent evaluator review, variant branching, or a heavier deliverable-quality loop.
- If the user asks about shortcuts, load `references/known-shortcuts.md`.

Do not bulk-load every reference file by default.

## Core operating rules

- Trust the live Paper MCP for the current tool surface; do not restate the whole schema here.
- Confirm the correct Paper page before creating/editing when page context is ambiguous.
- Prefer one artboard at a time unless the user explicitly wants a batch.
- Review your own Paper work by default before showing it.
- Use the `visual_reviewer` sub-agent for screenshot-based Paper critique.
- Keep canonical story panels in a vertical lane and put stuck-panel variants in a horizontal lane beside them.
- If a clear improvement is visible, do not stop to ask whether to keep going; continue autonomously until the panel is genuinely presentable or a real tradeoff/blocker appears.

## Keep this file lean

Detailed workflow, product caveats, icon rules, and evaluator patterns belong in the reference files, not repeated here.
